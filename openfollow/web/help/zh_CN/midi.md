# MIDI

本机的 USB MIDI 设备别名和虚拟推子源。配置后，patch 和推子值通过 MIDI 消息触发和推子占位符供给 OSC 发送器行。

## MIDI Patch

Patch 是一个稳定的编号槽位，将连接的 MIDI 设备分配给它——与 QLab MIDI Patch 的概念相同。Patch ID（而非端口名或别名）是其他所有内容引用的键（OSC 发送器触发、虚拟推子源）。ID 按顺序分配，在重启和设备交换间保持固定。

点击**添加新 MIDI Patch** 创建槽位，然后填写其行：

- **ID** – 只读永久整数键。被虚拟推子源和 OSC 发送器触发表单引用。
- **别名** – 可选的友好名称（最多 64 字符），如 `FOH` 或 `X-Touch`。在 patch 下拉菜单中与 ID 一起显示。
- **设备** – 绑定到此 patch 的 MIDI 输入端口。下拉列表列出当前连接的端口；已断开的设备显示为 `(未连接)`，保存保留绑定以便重新连接。

每行有其自己的**保存**和**删除**按钮。

> Patch ID `0` 保留为通配符，意为"任意 patch"——真实 patch 始终从 1 开始。

## 虚拟推子

八个归一化推子（0.00 – 1.00），按总线位置（1 到 8）寻址。每个推子的实时值显示在其条带中，通过推子占位符供给 OSC 发送器行。总线固定为八个；不能添加或移除条目。

点击条带在下方详情面板中打开其设置。

### 标识

- **显示名称** – 操作员屏幕（启用时）和引用此推子的触发表单中显示的标签。最多 32 字符；默认 `推子 1`、`推子 2` 等。
- **默认值** – 启动时推子的值（0.00 到 1.00）。
- **颜色** – 在 Web 界面中为推子条带着色的色块。
- **在操作员屏幕上显示** – 勾选后此推子出现在操作员屏幕上。

### 源

- **源类型** – `无源` 将推子保持在其默认值；`MIDI` 从传入 MIDI 消息驱动它。

**源类型**为 `MIDI` 时，显示以下额外字段：

- **Patch** – 监听哪个 MIDI patch。`(任意 patch)` 接受所有 patch 的消息。
- **消息类型** – 驱动此推子的 MIDI 消息：控制变更、按键压力或通道压力。
- **通道** – 要匹配的 MIDI 通道。`0` 匹配任何通道；`1`–`16` 针对特定通道。
- **CC / 音符号** – 要监听的控制号或音符号（0 – 127）。通道压力不使用。
- **学习** – 点击后在 10 秒内移动物理推子或旋钮。下一条传入消息自动填充 Patch、消息类型、通道和 CC/音符号。

点击**保存推子**应用所选推子的更改。

## 标记推子

只读。每个受控标记一个条带，由当前控制它的手柄驱动。从受控标记列表自动配置——此处无配置项。

标记推子的实时值通过 `[markerfader]` 占位符供给 OSC 发送器行。哪个手柄轴供给标记推子及其移动速度在手柄页面设置。
