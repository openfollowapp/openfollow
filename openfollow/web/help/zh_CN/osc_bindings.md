# OSC 发送器

配置发送到灯光控台、媒体服务器、音频引擎和演出控制软件的出站 OSC 消息。

每个发送器折叠为单行摘要（名称、触发类型、目标）。展开后显示四个标签页：**基本**、**触发**、**设置**、**诊断**。

## 基本

标识、目标和传输。

- **启用** – 此发送器的总开关；禁用的发送器不发送。如果占位符的依赖不满足（如裸 `[x]` 无默认标记），开关变为红色，发送器强制关闭直到解决。
- **名称** – 供自己参考的标签。显示在诊断标签页和总览实时计数器中。
- **默认标记** – 裸位置占位符（`[x]`、`[y]`、`[markerid]` 等）在未给定 `:N` 索引时解析的目标标记 ID。如果每个占位符都显式指定标记则留空；默认标记必须是已存在的标记（ID 从 1 开始）。
- **默认推子** – 裸推子占位符（`[fader]`，包括转换形式如 `[fader.pct]`）解析的目标虚拟推子。仅使用显式 `[fader:N]` 引用或无推子占位符时留为 `(无)`。
- **目标** – 此发送器发送到的共享 OSC 连接（主机、端口、传输），从 **OSC 目标** 下维护的列表中选择。无目标选择的发送器不发送任何内容。编辑目标的 IP 一次，所有指向它的发送器和区域实时重定向——无需逐行编辑、无需重启。下拉菜单下方的只读行显示目标解析后的 `host:port`。

## 触发

发送器何时发送。从下拉菜单选择一种**触发类型**；下方字段相应更新。

- **流** – 以选定**速率（Hz）**（1、5、10、20、30 或 60 Hz）连续发送。设置**发送**为：
  - *始终发送* – 无论是否移动，每次都触发。
  - *仅变化时发送* – 仅当发送器的默认标记自上次发送以来在任一轴上移动至少**最小变化（m）**后才触发。门控始终监视默认标记，即使消息体通过 `[x:N]` 引用其他标记。无默认标记的发送器每次都触发。

- **热键** – 在键盘组合上触发。设置**按键**、任意**修饰键**（Ctrl、Shift、Alt、Cmd）和**边沿**（按下或释放）。移动键被保留，不能分配。

- **手柄按钮** – 在手柄按钮边沿触发。设置**按钮**（A、B、X、Y、肩键、方向键、Start、Back 等）和**边沿**（按下或释放）。多个发送器可共享一个按钮独立触发。

- **MIDI 消息** – 当传入 MIDI 事件匹配模式时触发。所有匹配字段默认"任意"——留空匹配任何内容。
  - **类型** – 音符开、音符关、控制变更、程序变更、按键压力或通道压力。
  - **Patch** – 要匹配的 MIDI patch。`(任意)` 匹配全部。在输入 → MIDI 下配置 patch。
  - **通道** – 1–16 或 `Any`。
  - **编号** – 音符/CC/程序号（0–127），或留空任意。
  - **值** – 0–127，或留空任意。按 MIDI 规范，值为 0 的音符开视为音符关。
  - **捕获** – 点击后在 10 秒窗口内操作控件，从下一条传入 MIDI 消息自动填充所有字段。

- **推子变化** – 每当推子值变化时触发，按选定**速率（Hz）**限流，限制快速移动的推子产生网络流量的频率。从组合下拉菜单选择**推子**源，列出所有八个虚拟推子和每个标记的标记推子。

## 设置

此发送器发送的 OSC 消息——带可选占位符的地址和参数。

**OSC 消息** – 单个字段中的完整消息。第一个空格分隔的令牌是 OSC 地址；其余为参数。输入时识别的占位符显示为带标签的药丸；点击药丸编辑（如添加指向特定标记的 `:N` 索引）。使用字段下方的按钮在光标处插入占位符。

参数类型从字面量推断：整数（如 `12`）发送为 `i`；浮点数（如 `1.0`）发送为 `f`；其他为字符串 `s`。占位符自带类型（`[markerid]` → `i`；位置和推子占位符 → `f`）。含空格的参数必须引号括起：`"My Cue"` 是一个字符串参数，而非两个令牌。

**占位符语法。** 每个占位符遵循一种形式：

```
[ 源 (:N) (.转换) (.转换) … ]
```

- **`源`** – 要发送的值（见下）。
- **`:N`** – 可选索引：位置/`markerid`/`markerfader` 源的标记 ID，或 `fader` 的推子号。省略则按发送器的默认标记/默认推子解析。事件源不接受索引。`:cN` 形式为*控制器引用*（见下）。
- **`.转换`** – 可选过滤器链，从左到右应用。

**源：**

| 源 | 值 |
|---|---|
| `[x]` `[y]` `[z]` | 标记位置（米），X=舞台左，Y=舞台深处，Z=向上 |
| `[markerid]` | 解析标记的 ID（整数） |
| `[fader]` | 默认虚拟推子值，归一化 0.0–1.0 |
| `[markerfader]` | 解析标记自身的推子值，0.0–1.0 |
| `[value]` | 实时 MIDI 事件值（0–127）。仅 MIDI 消息发送器 |
| `[velocity]` | 音符力度（0–127）。仅音符开/关发送器 |
| `[note]` | MIDI 音符号（0–127）。仅音符开/关发送器 |

**索引 – 指定特定标记或推子。** 附加 `:N` 覆盖发送器默认值：`[x:2]` 发送标记 2 的 X（适用于 `x`/`y`/`z`/`markerid`/`markerfader`），`[fader:3]` 读取虚拟推子 3。发送器从该目标发送，不受默认标记/默认推子影响。

**控制器引用 – `:cN`。** 附加 `:cN`（从 1 开始：`c1` = 第一个控制器）来寻址*控制器 N 当前驱动*的标记，操作员移动时实时解析。用于告知控台操作员正在跟踪哪个灯具而无需硬编码标记 ID：`[markerid:c1]` 发送该标记的 ID，`[markerfader:c1.int:0-100]` 将其推子作为 0–100 发送，`[x:c1]`/`[y:c1]`/`[z:c1]` 发送其位置。仅对标记键控的源有效（不包括 `[fader]` 和事件源）；`c0` 不是控制器，渲染为字面文本。单个控制器连接时，`cN` 跟随当前选中的标记；多个时，为该控制器的固定标记。如果控制器断开或无驱动标记，发送器跳过该次发送（在诊断下显示）——不依赖发送器的默认标记。

**转换 – 用 `.` 链接，从左到右应用：**

| 转换 | 效果 | 适用于 |
|---|---|---|
| `.inv` | 反转——位置取负，0.0–1.0 推子取 `1 − v` | 位置、推子 |
| `.frac` | 按网格范围归一化到 ±1 | 位置 |
| `.pct` | 乘 100（0.0–1.0 → 0–100） | 推子 |
| `.int:min-max` | 缩放到**整数**范围，含上下限（`min > max` 反转方向） | 推子 |
| `.scale:min-max` | 缩放到**浮点**范围；上下限可为小数和负数 | 推子 |

位置（`[x]` `[y]` `[z]`）接受 `.inv`/`.frac`。推子（`[fader]` `[markerfader]`）接受 `.inv`/`.pct`/`.int`/`.scale`。`[markerid]` 和事件源不接受转换。`[z.frac]`（高度比例）额外需要设置**网格 → 最大高度**。

**示例：**

| 占位符 | 发送 |
|---|---|
| `[x.frac]` | X 作为网格比例，−1.0 … 1.0 |
| `[z.frac.inv]` | 反转高度比例 |
| `[fader.pct]` | 默认推子作为 0–100 |
| `[fader:3.int:0-255]` | 推子 3 缩放到 0–255 整数 |
| `[markerfader.scale:-60-12]` | 标记推子映射到 −60 … +12（如 dB） |
| `[x:2]` | 标记 2 的 X 位置 |
| `[markerid:c1]` | 控制器 1 当前驱动的标记 ID |
| `[markerfader:c1.int:0-100]` | 控制器 1 的标记推子，0–100 |

> 依赖尚未满足的占位符以红色显示。发送器不会启用，直到设置所需的默认标记/默认推子，或重写占位符使用显式引用。

## 诊断

只读面板，显示发送器当前正在做什么。

- **实时状态** – 连接状态（UDP 就绪，或 TCP 已连接/正在连接/退避中）、每秒数据包数和最后一次错误。点击刷新按钮更新。
- **预览** – 用当前标记数据渲染将要发送的消息，不实际发送。点击**刷新**重新渲染。
- **测试发送** – 点击**发送测试包**强制向目标发送一个包。即使发送器禁用也可工作，用于启用前验证目标和消息。结果立即显示。

## 添加和管理发送器

**模板**下拉菜单 + **+ 新建发送器** – 选择模板并点击按钮添加预填充其地址和参数的发送器。下拉菜单留空从空白发送器开始。拖拽折叠发送器左侧的 ⋮⋮ 手柄可重新排序——顺序仅为外观，发送器独立评估。

## 保存与共享

- **保存** – 将发送器当前设置写入磁盘。更改立即生效但不保存刷新后丢失。
- **放弃** – 将编辑器恢复到上次保存状态。需确认。其他打开的发送器不受影响。
- **保存为模板…** – 将发送器的名称和消息保存为可复用模板。显示在**模板**下拉菜单的*自定义模板*下。
- **复制** – 复制发送器并将副本插入正下方。适用于为第二个标记创建几乎相同的发送器。
- **删除** – 永久移除发送器。需确认。
