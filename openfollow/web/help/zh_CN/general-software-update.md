# 软件更新

安装更新的 OpenFollow 版本并重启服务——无需 SSH。

**已安装** 显示当前在此设备上运行的版本。

**检查并安装最新版** 查询 GitHub 获取最新发布的版本。如果比已安装版本更新，显示确认对话框；确认后下载此设备架构的签名更新包（`.ofupdate`），验证并安装。进度显示在锁定的对话框中，新版本启动后页面自动重载。如果设备已是最新，显示"已是最新"消息，不做任何更改。

安装过程脱离运行中的服务（软件包自行重启服务），因此设备会短暂离线，随后以新版本重新上线。如果更新程序缺少所需权限，安装失败并提示先在设备页面运行**应用权限**。

> **需要互联网。** 此选项访问 `api.github.com` 和发布下载。在无上行链路的隔离演出局域网中，请改用**离线安装**。

## 安装前验证内容

每次更新——在线或离线——在以 root 安装前都会检查，任何失败都将拒绝安装：

- **签名** – 更新包带有 OpenFollow 发布密钥的签名。设备持有匹配的公钥，拒绝任何未签名内容，防止被篡改或第三方包安装。
- **校验和** – 包的 SHA-256 必须与签名清单匹配，防止截断或损坏的下载被安装。
- **身份** – 包内必须是 `openfollow` 且匹配本设备架构。

## 离线安装

对于没有互联网的场地，展开**离线安装**来安装你提供的更新包。

- **选择文件** – 选择 `openfollow_<版本>_<架构>.ofupdate` 更新包，在可上网的机器上从发布页下载。
- **上传并安装** – 通过局域网上传，经签名 + 校验和验证后安装，与在线路径相同。

与在线检查不同，离线路径**不**对版本设限——只要签名正确，可以故意重装相同版本或降级。

> 更新包是包含 `.deb`、其校验和和签名的单个文件。只有为 OpenFollow 发布签名的 `.ofupdate` 才能安装——未签名或手动构建的包会被拒绝。
