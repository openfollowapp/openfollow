# 操作员消息

站点可以在**操作员屏幕**上显示简短文本消息——通常是操作员的下一条提示——由控台或演出控制系统通过 OSC 推送。它们与 **OSC 输入**共享同一个接收器，因此必须在那里**启用** OSC 服务器才能接收任何消息。

## 设置

- **启用** – 总开关。关闭时，忽略传入的操作员消息，操作员屏幕上不绘制任何内容。默认：关闭。
- **位置** – 消息卡片堆叠的显示位置：**底部居中** 或 **顶部居中**。
- **最大可见数** – 同时显示的卡片数量，1–20。默认：`5`。超出此数量的卡片折叠为一行"+N 更多"，确保叠加层不会超出屏幕。
- **按标记路由** – 开启时（默认），标记键控消息（`markerId` ≥ 1）仅显示在控制该标记的站点上。关闭则在所有站点上显示每条消息，不论 `markerId`。广播消息（`markerId` 0）始终到达所有站点。
- **缩放** – 统一放大消息卡片及其文字：`1×`、`1.25×`、`1.5×`、`1.75×` 或 `2×`。默认：`1×`。卡片宽度仍限制在屏幕的 60%，因此较长的行会换行而不会在较大尺寸下超出屏幕。

## 发送消息

发送到地址，最多四个位置参数（OSC 类型标签 `ssif`）：

```
/message  <消息:s>  <信息:s>  <标记ID:i>  <秒数:f>
```

- **message** – OSC **字符串** (`s`)。主文本行。
- **info** – OSC **字符串** (`s`)。可选的第二行（如提示编号）。默认为空。
- **markerId** – OSC **整数** (`i`)。路由目标（见下）。默认 `0`。
- **seconds** – OSC **浮点数** (`f`)。此秒数后自动消除。`0` 保持卡片直到手动清除。默认 `0`。

后续参数从左到右可选填充；无法读取的数字参数会丢弃整条消息。数字宽松读取——`markerId`/`seconds` 也可以数字字符串（`"3"`、`"8.0"`）或其他数字类型传入——但 `i` / `f` 是标准类型。

## 按标记路由

- **`0` – 广播。** 在所有站点上显示。广播卡片无标题栏。
- **`1` 或以上 – 标记路由。** 仅显示在当前控制该标记的站点上；其他所有站点丢弃。卡片带有标记颜色的实心标题栏，显示标记名称。

这使得一个发送端可以寻址"控制标记 3 的人"，而无需知道是哪个物理站点。关闭**按标记路由**（设置中）可忽略此规则，在所有站点上显示每条消息。

## 清除

```
/message/clear            清除所有消息
/message/clear  <id>      清除标记 <id> 的消息
```

你也可以在**输入**标签页中绑定按键或手柄按钮来清除本站点的消息。

## 传输

消息通过 **OSC 输入**中配置的 OSC 接收器传输——直接单播到站点、子网广播，或使用那里设置的**组播组**一次性到达所有站点。
