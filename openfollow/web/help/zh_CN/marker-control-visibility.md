# 标记控制与可见性

定义演出中存在哪些标记以及本机如何驱动和显示它们。**目录**在局域网上每个 OpenFollow 站点间共享；**选择**是本机独有的。

## 共享目录

所有标记的实时表格，保存后几秒内同步到所有站点。每行：

- **ID** – 数字标识符（整数 ≥ 1），接收端在 PSN、OSC、OTP 和 RTTrPM 上看到。匹配控台的编号。
- **名称** – 界面的人类可读标签；不在 PSN 线路上发送。
- **颜色** – 标记在操作员屏幕和 Web UI 中的色块；点击打开选择器。添加行时建议使用第一个未使用的调色板颜色。
- **控制方/查看方** – 只读：哪些站点当前声明控制或查看。⚠ 和红色行高亮标记控制冲突（多个站点控制同一标记）；在各站点的选择表中解决。
- **保存** / **删除**（每行） – 提交行的名称/颜色，或从所有站点移除标记（经确认提示后）。

通过底行（**ID**、**名称**、**颜色** → **添加**）添加标记；ID 预填下一个空闲整数。

> 删除其他站点仍选中的标记会在下次同步时从该站点移除——演出中删除目录条目前请协调。

## 本机选择

本机驱动和渲染哪些标记。更改立即生效并自动保存（无保存按钮）。

- **控制** – 声明控制：路由到本机的输入（手柄、键盘、鼠标、OSC）可驱动标记，且该标记包含在本机的 PSN 广播中。
- **查看** – 在操作员屏幕上渲染标记，无论哪个站点控制它。

两者按行独立——可查看而不控制，或控制而不渲染。
