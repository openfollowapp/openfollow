# 诊断

Web 服务器、对等发现和日志的实时运行时健康状况。使用此区域一目了然地发现故障子系统，并下载或查看支持报告所需的信息。

该区域位于总览标签页内，每五秒刷新一次。

## 实时摘要卡片

区域顶部四张卡片。每张卡片显示状态标签和少量指标。

### Web 服务器

- **已配置** – 配置中设置的 Web 端口。
- **服务端口** – 服务器实际绑定的端口。如果 80 端口不可用，OpenFollow 回退到备用端口（8080 或 2010），此行显示使用的端口。
- **运行时间** – OpenFollow 进程已运行的时间。
- 状态标签：配置端口与实际端口匹配时显示 **OK**，使用备用端口时显示**回退**。

### 信标发送器

出站对等发现信标的健康状况。如果此卡片不健康，网络上的其他站点将看不到本机。

- **上次发送** – 最近一次出站信标的时间戳。
- **错误数** – 进程启动以来的发送错误累计。
- **发送总数** – 进程启动以来发送的信标总数。

### 信标接收器

入站对等发现监听器的健康状况。如果此卡片不健康，本机将看不到其他站点。

- **已发现对等站点** – 本地网络上当前已知的 OpenFollow 站点数量。
- **最近数据包** – 最近收到信标的时间戳。
- **数据包总数** – 进程启动以来接收器处理的数据包累计。

### 日志

告诉你日志行来自哪里。正常情况下显示 **journalctl**（系统日志，首选）。如果系统日志不可达，回退到 **ring**——覆盖当前会话的内存日志缓冲区。

> 如果预期应为 journalctl 但不可用，卡片上方会显示黄色警告横幅说明原因。

## 打包与工具

卡片下方有四个操作。

- **下载诊断包** – 生成单个 UTF-8 文本文件并保存到浏览器。包为纯文本，易于浏览，包含排查故障安装所需的一切：Web 服务器和发现状态、完整脱敏配置、最近的错误、日志尾部、运行时和依赖版本、操作系统和硬件详情、网络接口、USB 设备树等。Web PIN 始终替换为 `***`；认证签名值从日志行中剥离。公开分享前请检查，因为其中包含 IP、主机名和 USB 描述符原样。

- **测试对等连接** – 在其通告端口上探测每个已知站点并显示结果小表格。绿色标签表示站点响应了 HTTP 状态和往返时间；红色标签表示不可达，附带原因。如果表格显示"暂无已知对等站点"，等待发现或检查上方**信标接收器**卡片。

- **显示最近日志尾部（最近 100 行）** – 从实时日志源拉取最近 100 行并内联显示。第一行标明来源。认证签名在显示前已脱敏。

- **重启应用** – 重启 OpenFollow 进程（非操作系统）。首先显示确认提示；服务器恢复后页面自动重载。在更改需要重启的设置后使用，如 Web 端口或人形检测引擎。
